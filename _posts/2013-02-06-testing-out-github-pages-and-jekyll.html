Github Pages

If you have never heard about Github pages before and are looking for somewhere to host your own personal blog, then I would suggest checking it out now. pages.github.com host static html websites using your own repository to manage the files. You get one site per GitHub account and unlimited project sites. Simply create a new repository named [username].github.io and you are away!

Once you have your repository, you can then start to create static html pages, e.g. index.html that will then display when navigating to [username].github.io

Blogging with Jekyll

Not only does GitHub Pages support static html pages, but it also has support for Jekyll

Jekyll is a blog-aware, static site generator in Ruby (http://jekyllrb.com)[http://jekyllrb.com]

Using Jekyll to generate your static html pages makes managing your website layout/headers/footers/menus much easier than manually editing multiple html files. Along with generating html pages from templates, it also makes blogging a lot easier. Jakyll will take your blog post content pages, the layout pages, and various other pages, process them when you push your files up to your repository and then output all the static pages.
